# ⚡ התחלה מהירה - רשימת המתנה

## 🎯 בעיה: "לחיצה על רשימת המתנה שולחת לעמוד הבית"

זו בעיה נפוצה - **הפתרון פשוט:**

### 1️⃣ רענן את האפליקציה **לגמרי**

```bash
# עצור את השרת (לחץ Ctrl+C בטרמינל)

# רענן עם ניקוי cache
npx expo start -c
```

או אם זה לא עובד:
```bash
npm start -- --reset-cache
```

### 2️⃣ אחרי שהשרת עולה - רענן את האפליקציה במכשיר

- **iOS Simulator:** לחץ `Cmd + R`
- **Android Emulator:** לחץ `R + R` (פעמיים מהר)
- **טלפון פיזי:** נער את הטלפון ובחר "Reload"

### 3️⃣ נסה שוב

עכשיו כשתלחץ על "רשימת המתנה" זה אמור לעבוד! ✅

---

## 📋 מה עוד צריך לעשות?

### כן! צריך ליצור אינדקסים ב-Firestore

אחרת תקבל שגיאות כשתנסה להשתמש ברשימת המתנה.

#### הדרך הקלה:
1. הרץ את האפליקציה
2. כאדמין, נסה להיכנס למסך "רשימת המתנה"
3. תופיע שגיאה בקונסול עם **קישור**
4. **לחץ על הקישור** - Firebase תיצור את האינדקס בשבילך
5. חכה 2-5 דקות
6. רענן את האפליקציה

**זהו! זה הכל!** 🎉

---

## ✅ בדיקה מהירה שהכל עובד

### כאדמין:
1. ניהול אדמין → גלול מטה
2. אתה אמור לראות כרטיס אדום: "רשימת המתנה"
3. לחץ עליו
4. אתה אמור לראות מסך עם לוח של 7 ימים

### כלקוח:
1. הזמנת תור → בחר ספר, טיפול, תאריך
2. גלול מטה
3. אתה אמור לראות מלבן אדום: "לא מצאת תור לזמן שלך?"
4. לחץ עליו
5. בחר טווח שעות והירשם

---

## 🆘 עדיין לא עובד?

### אם לוחץ על "רשימת המתנה" ונזרק לעמוד הבית:

**סימן 1: אתה לא אדמין**
- הרץ סקריפט להפיכתך לאדמין
- או בדוק שהמשתמש שלך יש לו `isAdmin: true` ב-Firestore

**סימן 2: הקובץ לא נטען**
```bash
# נקה הכל ונסה שוב
rm -rf node_modules/.cache
npx expo start -c
```

**סימן 3: בעיה בניווט**
- בדוק בקונסול אם יש שגיאות
- ודא שהקובץ `AdminWaitlistScreen.tsx` קיים בתיקייה `app/screens/`
